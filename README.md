# 就活管理アプリ

就職活動で発生する企業情報、選考状況、締切、面接予定、ES、面接対策メモを一元管理するためのアプリです。

公開URL: https://shukatsu-tracker-nine.vercel.app

## 作成物の説明

就活では、企業ごとにマイページ、ES締切、面接日程、説明会、選考ステータス、志望度、提出内容などが分散しやすく、管理漏れが起きやすいという課題があります。

このアプリでは、企業ごとに以下の情報をまとめて管理できます。

- 企業名、業種、志望度、選考種別
- 本選考、インターン、早期選考などの分類
- 締切、面接、説明会などの予定
- 採用マイページURL、マイページID
- ESの質問と回答
- 回答文字数の自動カウント
- 面接対策メモ、企業メモ
- 選考中、通過、内定、落選などのステータス
- 企業アイコン、企業公式サイトへのリンク
- バックアップと復元

PCとスマートフォンの両方で使えるクラウド同期版に加えて、Windows/Macでローカルに動かせる配布版も用意しています。

## 直面した課題と解決方法

### PCとスマートフォンで同じデータを使いたい

最初はブラウザ内の保存だけで動く形にしましたが、それだけでは別端末と同期できませんでした。

そこでSupabaseを導入し、ログインしたユーザーごとに企業情報、予定、ES、テンプレートをクラウド保存できるようにしました。

### 他のユーザーに使わせるときのプライバシー

就活情報には企業名、ES、面接メモなどの個人的な情報が含まれます。

そのため、Supabase側ではRow Level Securityを有効化し、ログイン中のユーザーが自分のデータだけを読み書きできる設計にしました。

また、クラウドを使いたくない人向けに、端末内だけにデータを保存するローカル版も用意しました。

### ESを実用的に管理したい

ESは単にメモとして保存するだけでは使いにくいため、質問と回答を分けて保存できるようにしました。

さらに、回答文字数を自動で表示し、ガクチカ、自己PR、志望動機などの使い回せる文章テンプレートを別ページで管理できるようにしました。

### 継続して使いたくなるUIにしたい

単なる管理表では使うモチベーションが下がるため、マスコットキャラクター、今日の格言、お祝い演出、落選時の励まし表示などを追加しました。

選考通過や内定のときには派手なお祝いを表示し、落選時には「なんて見る目のない企業なの！！」というメッセージを表示するようにしています。

### アーキテクチャ

フロントエンドはHTML、CSS、JavaScriptで構成しています。

クラウド同期版では、ブラウザからSupabaseに接続し、ログインユーザーごとにデータを保存します。SupabaseのRow Level Securityにより、ユーザーごとのデータ分離を行っています。

ローカル配布版では、Supabaseに接続せず、ブラウザのローカル保存を使って端末内にデータを保持します。これにより、サーバーへ就活情報を送らずに利用できます。

## ローカル配布版

クラウド同期版とは別に、Windows/Macで使えるローカル配布版も作成しています。

ローカル配布版ではSupabaseに接続せず、就活データは利用者の端末内に保存されます。PCを買い替える場合や別端末へ移したい場合は、アプリ内のバックアップ・復元機能でJSONファイルを書き出して移行できます。

配布用zipはGitHub Releasesに添付して配布する想定です。

### 使用モデル

アプリ内で外部AIモデルは使用していません。使い方の案内はFAQ形式で実装しています。

### 実装方法

企業、予定、ES、テンプレートをJavaScript側で状態管理し、画面に反映しています。クラウド同期版ではSupabaseのテーブルに保存し、ローカル版では端末内の保存領域を使います。

バックアップ機能では、登録データをJSONファイルとして書き出し、別の環境で復元できるようにしています。

## 個人情報への配慮

このリポジトリには、実際の就活情報、個人のES本文、ログイン情報、パスワードなどは含めていません。

Supabaseの公開用キーはフロントエンドで利用する前提のキーであり、データの閲覧・更新範囲はRow Level Securityで制御しています。

## セキュリティ面

- GitHubにはSupabaseの `service_role` key、データベースパスワード、個人のログインパスワードは含めていません。
- ブラウザ側に置いているのは、公開して使う前提のSupabase Publishable keyです。
- `entries` と `es_templates` の各テーブルではRow Level Securityを有効化し、`auth.uid()` と `user_id` が一致するデータだけを読み書きできるようにしています。
- 未ログインユーザーや別ユーザーのデータは、RLSの条件に合わないため取得・更新できない設計です。
- 企業名、ES、メモなどの入力内容は、画面表示時にHTMLとして実行されないようにエスケープ処理をしています。
